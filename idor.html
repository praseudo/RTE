Insecure Direct Object Reference

### Description:

Insecure direct object references (IDOR) are a type of access control vulnerability that arises when an application uses user-supplied input to access objects directly. In such cases, the attacker can manipulate those references to get access to unauthorized data.

### Mitigation:

There is essentially a data-contextual access control enforcement need. A URL or even a POSTed form should NEVER contain an access control "key" or similar that provides automatic verification. A data contextual check needs to be done, server side, with each request.
